# Data Directory

## توضیحات

این پوشه شامل داده‌های خام، پردازش‌شده و فایل‌های داخلی سیستم است.

## ساختار

### `transactions.jsonl`

فایل اصلی حاوی تراکنش‌های مالی به فرمت JSON Lines.

**فرمت:**
هر خط یک تراکنش کامل به صورت JSON است:

```json
{"transaction_id": "...", "timestamp": "...", "amount": 10000, ...}
```

**حجم:** ~20,000+ تراکنش

### `chunks_head/`

پوشه حاوی chunk های داده برای پردازش موازی.

**فایل‌ها:**

- `000001`, `000002`, `000003`: بخش‌های تقسیم‌شده داده

### `wal/`

Write-Ahead Log files برای consistency و recovery.

**فایل‌ها:**

- `00000000` - `00000003`: Log های تغییرات

### `queries.active`

فایل کوئری‌های فعال سیستم.

## نکات مهم

⚠️ **توجه:**

- این پوشه حاوی داده‌های تولیدشده توسط سیستم است
- فایل‌های WAL نباید دستی تغییر داده شوند
- برای backup، تمام محتوای این پوشه را کپی کنید

## پاکسازی داده‌ها

برای پاکسازی و شروع مجدد:

```bash
# حذف تمام داده‌ها (احتیاط!)
rm -rf chunks_head/* wal/* queries.active

# نگهداری transactions.jsonl اصلی
# این فایل را حذف نکنید مگر اینکه بخواهید از نو شروع کنید
```

## حجم فایل‌ها

- `transactions.jsonl`: ~50-100 MB
- `chunks_head/`: ~10-20 MB
- `wal/`: ~5-10 MB
